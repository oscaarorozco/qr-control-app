# Contributing

Thanks for your interest in contributing.

## Development setup

1. Fork and clone the repository.
2. Copy `.env.example` to `.env.local`.
3. Install dependencies: `npm install`.
4. Start development server: `npm run dev`.

## Pull request guidelines

1. Keep changes focused and small.
2. Include tests or validation steps when applicable.
3. Run `npm run lint` and `npm run build` before opening a PR.
4. Do not commit secrets, real databases, or personal data.
5. Keep UI/content generic in demo fixtures.

## Commit style

Use clear and descriptive commits:

- `feat: add component image upload`
- `fix: enforce admin scope check in activity route`
- `docs: update open source setup section`

## Security

If you find a security issue, please follow `SECURITY.md` instead of opening a public issue.
